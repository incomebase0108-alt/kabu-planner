# 日本株 売買プランナー

`app.html` を**ブラウザで開くだけ**で動きます（スマホ可）。サーバー不要。

- `app.html` … ツール本体。**これを開く**
- `index.html` … 紹介用のランディングページ（ツール本体ではない）

公開版: https://kabu.incomebase.jp

## 使い方
1. 銘柄コード（例: `7203` トヨタ、`186A` アストロスケール）を入れて「実行」
2. スタイル（短期／中期／長期）を選ぶと利確・損切りの幅が変わる
3. 表示されるもの：
   - 現在値・前日比
   - 短期／長期トレンド判定（移動平均線ベース）
   - RSI（買われすぎ／売られすぎ）
   - **買い目安・利確ライン・損切りライン**（ATR＝値動きの大きさから自動算出）
   - リスク／リワード比

株価取得に失敗したときは「手動で現在値を入れる」を開き、現在値（＋任意で直近高値・安値）を
入れれば利確・損切りは計算できます。

# 改造して使う人へ

## 最初にやること：自分用のプロキシWorkerを立てる

株価は Yahoo Finance の公開APIから取りますが、ブラウザから直接叩くとCORSで弾かれます。
そのため中継用の Cloudflare Worker を経由します。

**既定値は作者のWorkerを指しているので、必ず自分用に立て直してください**
（`app.html` の `DEFAULT_PROXY`）。作者の無料枠を消費してしまいます。

1. https://dash.cloudflare.com でアカウント作成（無料）
2. Workers & Pages → Create Worker
3. このリポジトリの `cloudflare-worker.js` の中身を**全部貼り付けて** Deploy
   - 環境変数の設定は不要。`wrangler.toml` も不要
   - 中身の書き換えも不要（許可ドメインは汎用的に書いてある）
4. 発行された `https://xxx.workers.dev` をコピー
5. `app.html` を開き、**接続設定**の「自前プロキシURL」欄に貼って保存
6. 同じ画面の**診断**ボタンで各経路をテストし、「自前Worker」が成功すればOK

**ソースの書き換えは不要です。** 設定はブラウザの localStorage（`toushi_proxy`）に保存され、
以後この経路が最優先で使われます。

Worker がやっているのは、Yahoo Finance と Stooq への中継のみです
（ホワイトリスト方式。他のドメインは中継しない）。レスポンスは30秒キャッシュします。

## 踏みやすい罠

### 1. 銘柄辞書 `window.STOCKS` が二重に存在する

| 場所 | 状態 |
|------|------|
| `app.html:482` インライン | **こちらが実際に動いている** |
| `stocks.js` | どこからも読み込まれていない（`<script src>` が無い） |

`stocks.js` を直しても**何も起きません**。動作に効くのは `app.html` インラインの方です。
ただし後述の更新スクリプトが両方を同時に書き換えるため、内容は同期しています
（現在どちらも4,157銘柄）。片方だけ手で直すと差が生まれるので、手編集は避けてください。

### 2. 銘柄辞書は新規上場に追従しない

JPX公式データのスナップショットです。新しい銘柄を扱うには再生成します。

```bash
pip install xlrd
python tools/update_stocks.py              # JPXから自動ダウンロードして更新
python tools/update_stocks.py path/to.xls  # ローカルのxlsを指定
```

`stocks.js` と `app.html` の**両方**を更新します。月1回程度が目安です。
銘柄コードの検証は `/^[0-9][0-9A-Z]{3}$/`（`186A` のような英字入りコードに対応済み）。

### 3. 起動時に4.8秒の演出が入る

`app.html:1370` 付近。ターミナル風のブートログを表示してからフェードアウトします。
**自動テストやヘッドレスでの検証時はこれが邪魔になります**（画面が出るまで待たされる）。
演出を消すなら該当箇所とCSSの `bootout` アニメーション（`app.html:193`）を触ってください。

### 4. 状態は全部 localStorage にある

| キー | 内容 |
|------|------|
| `toushi_watch` | ウォッチリスト |
| `toushi_holds` | 保有株 |
| `toushi_alerts` | アラート設定 |
| `toushi_proxy` | 自前プロキシURL |
| `toushi_snd` | 音声通知 |
| `toushi_scan` | スクリーニング結果キャッシュ |

サーバーもDBもありません。バックアップ／復元機能が画面内にあります
（`toushi_scan` はキャッシュなのでバックアップ対象外）。

**キャッシュ絡みの注意**: スクリーニング結果は上位N件だけを `toushi_scan` に保存します。
スコアが下位に来るカテゴリ（例: 売り推奨はスコアが必ずマイナス）を新しく追加すると、
**リロード後に消える**という偽陰性バグを踏みます。過去に実際に踏みました。

## 構成

単一ファイル構成です。ビルド不要、依存パッケージ無し、`npm install` もありません。

```
app.html            ツール本体（約267KB・HTML/CSS/JSが全部入り）
index.html          ランディングページ
stocks.js           銘柄辞書（死にコード。更新スクリプトの出力先の一つ）
cloudflare-worker.js プロキシWorkerのソース
tools/update_stocks.py  銘柄辞書の再生成スクリプト
```

## 注意
- これは過去の株価から機械的に計算した**目安を表示する道具**であり、値上がり・利益を**保証しません**。
- 銘柄が上がるかどうかは誰にも当てられません。損切りを機械的に守るための補助として使ってください。
- 投資は自己責任です。
- 株価データは Yahoo Finance の非公式APIに依存しており、遅延・欠損・予告なく停止する可能性があります。
